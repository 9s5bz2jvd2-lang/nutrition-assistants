# RUNBOOK — cspi-uncompromised-dga demand-sensitive reading

Core rule:

> Start sparse. Expand on demand. Always run missed-case sweep for special populations and medical boundaries.

## 0. Before you read

1. What is the task type?
   - quick lookup (e.g., "what's a cup equivalent?")
   - dietary pattern calculation
   - guideline interpretation
   - alcohol/sustainability deep dive
2. Is the user asking about the official DGA or the uncompromised version?
3. Is a special population involved (infant, child, pregnant/lactating)?

## 1. Reading modes

| Mode | Read first | Expand when | Stop when |
|---|---|---|---|
| quick lookup | `SKILL.md`, one expert | answer unclear | answer bounded and sourced |
| dietary pattern | `guidelines-overview.md`, `dietary-pattern.md` | calorie level or age group unclear | quantitative answer with age/calorie qualification |
| alcohol question | `guidelines-overview.md`, `alcohol-context.md` | user asks about "official" limits | both versions clearly stated with context |
| sustainability | `guidelines-overview.md`, `sustainable-diet.md` | environmental + health intersection | 5 practices stated with cross-references |
| full audit | all modules | always | new version validated |

## 2. Missed-case sweep (always)

- [ ] Special population check (infant/child/pregnant/lactating/older adult)
- [ ] Medical advice boundary (population guidance ≠ clinical treatment)
- [ ] Official vs. uncompromised clarification (especially alcohol)
- [ ] Age-group specification (children < 2, < 14 have different limits)
- [ ] Evidence source labeling (DGAC, CSPI, FAO/WHO)
